## 1. 概述

### 1.1 文档目的

本文档定义OperatorBoard系统的测试策略、测试设计、测试用例和验证标准，确保系统功能、质量和用户体验达到预期要求。

### 1.2 测试范围

| 测试类型 | 覆盖范围 |
|---------|---------|
| 单元测试 | Python Agent Framework, Java Service, React组件 |
| 集成测试 | Agent间通信, API接口, 数据库交互 |
| E2E测试 | 完整用户流程, 跨系统交互 |
| 性能测试 | API响应, 并发处理, 资源占用 |

## 2. 测试策略

### 2.1 测试金字塔

```
                    ┌───────────┐
                    │   E2E     │  10%
                    │   Tests   │
              ┌─────┴───────────┴─────┐
              │     Integration       │  30%
              │       Tests          │
        ┌─────┴───────────────────────┴─────┐
        │          Unit Tests                │  60%
        │  (Python + Java + React)          │
        └─────────────────────────────────────┘
```

### 2.2 测试技术栈

| 层级 | 测试框架 | 工具 |
|------|---------|------|
| Python单元 | pytest | pytest-asyncio, pytest-cov |
| Java单元 | JUnit 5 | Mockito, AssertJ |
| React组件 | Vitest | React Testing Library |
| E2E | Playwright | @playwright/test |
| 集成 | requests | httpx |

### 2.3 测试环境

| 环境 | 用途 | 数据 |
|------|------|------|
| local | 开发调试 | 少量测试数据 |
| dev | 集成测试 | 全量脱敏数据 |
| staging | 预发布验证 | 生产镜像数据 |
| prod | 最终验收 | 真实数据(部分) |

## 6. 测试执行

### 6.1 测试命令

```bash
# Python单元测试
cd src/agent-framework && pytest

# Python集成测试
cd src/operator-agent && pytest tests/integration/

# Java单元测试
cd operator-service && mvn test

# E2E测试
cd agent-app && npx playwright test --project=chromium --reporter=line

# 完整测试套件
npm run test:all
```
